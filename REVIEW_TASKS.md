# Sonata 1.3 代码审阅与测试任务清单

## 审阅目标
确保数据源100%真实联网，修复所有问题，达到可交付标准。

## 模块分工

### 模块A：蒙特卡洛模拟预测
**负责人**: 资深股民Agent (InvestorAgent)
**范围**:
- 蒙特卡洛算法逻辑正确性
- 概率计算准确性
- 情景分析合理性
- 数据源真实性验证
- 预测结果可信度

### 模块B：热门板块 & 精选股票池
**负责人**: 项目工程师Agent (EngineerAgent)
**范围**:
- 热门板块发现算法
- 精选股票筛选逻辑
- 资金流数据真实性
- API调用可靠性
- 数据标注准确性

## 检查清单

### 数据源真实性
- [ ] 所有数据来自真实API（东方财富等）
- [ ] 无模拟/硬编码数据混入
- [ ] 数据来源明确标注
- [ ] API失败时正确处理（不返回模拟数据）

### 算法正确性
- [ ] 蒙特卡洛模拟次数足够（≥10000次）
- [ ] 概率计算无逻辑错误
- [ ] 股票评分算法合理
- [ ] 权重配置正确

### 代码质量
- [ ] TypeScript类型完整
- [ ] 无console.log调试代码
- [ ] 错误处理完善
- [ ] 性能优化到位

### 用户体验
- [ ] 数据加载状态清晰
- [ ] 错误提示友好
- [ ] 结果展示直观

## 交付标准
1. 100%真实数据源
2. 算法逻辑正确
3. 代码无已知Bug
4. 用户体验流畅
5. 文档完整
