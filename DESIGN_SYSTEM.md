# Danny Road 设计系统 - 专业金融终端配色规范

## 🎨 色彩哲学

参考 Bloomberg Terminal、Wind 金融终端的设计理念：
- **克制用色**：减少颜色数量，每种颜色都有明确语义
- **专业稳重**：避免花哨渐变，使用简洁纯色
- **信息优先**：重要信息通过对比度和位置突出，而非鲜艳颜色
- **一致性**：全系统统一色彩语言

## 🔑 关键信息视觉优先级

### 优先级 1（最高）- 操作建议
- **背景**：`#1E40AF`（深蓝纯色，去除渐变）
- **文字**：`#FFFFFF` 白色
- **样式**：圆角 `8px`，内边距 `16px`
- **位置**：量化总结区域顶部

### 优先级 2 - 关键价位
- **买入价**：蓝色系 `#1E40AF` 文字 + `#DBEAFE` 背景
- **止损位**：红色系 `#DC2626` 文字 + `#FEE2E2` 背景
- **止盈位**：绿色系 `#059669` 文字 + `#D1FAE5` 背景

### 优先级 3 - 量化评分
- **高分（≥60）**：绿色 `#059669`
- **低分（<60）**：红色 `#DC2626`

---

## 🚫 去除的设计元素

1. **渐变背景**：`bg-gradient-to-r from-indigo-500 to-purple-600` → 纯色 `bg-[#1E40AF]`
2. **过度阴影**：`shadow-lg` → `shadow-sm` 或 `shadow`
3. **花哨动画**：移除不必要的 `animate-` 效果，保留基础 `fade-in`
4. **过多颜色**：每个卡片背景色从 5-6 种减少到 2-3 种

## 📝 使用原则

1. **60-30-10 法则**：60% 中性色，30% 主色，10% 强调色
2. **语义优先**：颜色必须传达明确含义（涨/跌/警示）
3. **可访问性**：确保色盲用户可通过文字/图标区分
4. **一致性**：相同语义的元素使用相同颜色
