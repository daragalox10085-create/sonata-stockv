## 🐛 发现的 Bug

### BUG-001: K 线数据源覆盖不足 【P0 - 严重】

**问题描述**:  
贵州茅台（600519）等部分股票 K 线数据返回 0 条，但实时价格正常。

**影响范围**:  
- 高价股（>1000 元）
- 部分大盘股

**根本原因**:  
腾讯 K 线 API 对某些股票返回数据格式不一致或空数据。

**复现步骤**:
1. 打开 http://localhost:5177
2. 点击"贵州茅台"
3. 点击"开始分析"
4. 观察 K 线图显示"0 条真实数据"

**控制台日志**:
```
[K 线 API] 双源数据均失败
```

**解决方案**: ✅ 已修复
- 添加东方财富作为第三数据源
- 优先级：腾讯 > 东方财富 > 新浪
- 使用 CORS 代理解决新浪跨域问题

**修复代码**:
```typescript
// 三源交叉验证
const [tencentResult, sinaResult, eastmoneyResult] = await Promise.allSettled([
  fetchTencentKLine(code, days),
  fetchSinaKLine(code, days),
  fetchEastmoneyKLine(symbol, days)
]);
```

### BUG-002: 量能评分为 0 分 【P1 - 中等】

**问题描述**:  
贵州茅台的量能评分显示 0 分，但成交量数据正常（3 万手）。

**影响范围**:  
- 所有股票的五维决策引擎

**根本原因**:  
量能计算公式可能存在除零错误或数据映射问题。

**复现步骤**:
1. 分析任意股票
2. 查看五维决策引擎
3. 量能维度显示 0 分

**解决方案**: 🔧 待修复
- 检查量能计算公式
- 添加默认值保护
- 确保分母不为零

### BUG-003: 趋势判断与实际情况不符 【P2 - 轻微】

**问题描述**:  
贵州茅台显示"下降趋势中，均线系统呈空头排列"，但实际股价上涨（+0.21%）。

**影响范围**:  
- 技术面分析的趋势维度

**根本原因**:  
趋势判断逻辑可能基于历史数据而非实时数据。

**解决方案**: 🔧 待修复
- 优化趋势判断算法
- 结合短期和长期均线
- 考虑加入价格动量因子
