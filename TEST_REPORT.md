## 4. 发现的Bug和问题列表

### 4.1 高优先级问题

| ID | 问题描述 | 影响 | 复现步骤 |
|----|----------|------|----------|
| BUG-001 | K线数据API获取失败 | 高 | 搜索任意股票，查看控制台报错 |
| BUG-002 | 热度评分显示为NaN | 中 | 查看热门板块区域 |

### 4.2 中优先级问题

| ID | 问题描述 | 影响 | 复现步骤 |
|----|----------|------|----------|
| BUG-003 | ECharts实例dispose警告 | 低 | 频繁切换股票时查看控制台 |

### 4.3 问题详情

#### BUG-001: K线数据API获取失败
```
错误信息: [腾讯财经 API] 获取 K 线数据失败: SyntaxError: Unexpected token '<', "<!doctype "... is not valid JSON
错误信息: [K 线 API] 所有时间周期数据均失败
位置: StockContext.tsx:422, StockContext.tsx:449
```

**可能原因**:
- 腾讯财经API返回了HTML错误页面而非JSON数据
- API端点可能已更改或需要认证
- 网络代理问题

**建议修复**:
1. 检查API端点是否正确
2. 添加错误处理和重试机制
3. 考虑使用备用数据源

#### BUG-002: 热度评分显示为NaN
```
位置: 热门板块区域
表现: 热度评分显示为 "NaN"
```

**可能原因**:
- 后端返回的数据格式不正确
- 前端数据解析逻辑有问题

**建议修复**:
1. 检查后端API返回的数据格式
2. 添加数据验证和默认值处理

#### BUG-003: ECharts实例dispose警告
```
警告信息: [ECharts] Instance ec_xxx has been disposed
位置: echarts.js:7472
```

**可能原因**:
- 组件卸载时ECharts实例未正确销毁
- 频繁切换股票时图表重新初始化

**建议修复**:
1. 确保组件卸载时正确销毁ECharts实例
2. 使用useEffect清理函数
