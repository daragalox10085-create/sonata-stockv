### 3. 盈亏比显示验证测试

**测试场景**: 测试无效输入时盈亏比的显示

**测试结果**: ✅ **通过**
- 正常情况：显示 "1 : 3.23" (513310) 和 "1 : 3.33" (600519)
- 无效情况处理：当 risk <= 0 或 profit <= 0 时，显示 "无效"
- 计算逻辑正确：`ratio = (takeProfit1 - buyPrice) / (buyPrice - stopLoss)`

**验证代码**:
```typescript
const calculateRiskRewardRatio = (): string => {
  const profit = displayLevels.takeProfit1 - displayLevels.buyPrice;
  const risk = displayLevels.buyPrice - displayLevels.stopLoss;
  
  if (risk <= 0) return '无效';
  if (profit <= 0) return '无效';
  
  const ratio = profit / risk;
  if (!isFinite(ratio)) return '无效';
  
  return ratio.toFixed(2);
};
```

**描述**: 当手动输入负数时，HTML input type="number" 会自动移除负号，但 TypeScript 验证逻辑仍然存在。建议在前端添加更明显的错误提示。  
**现状**: 虽然有验证逻辑，但用户体验不够直观（负号被静默移除）

